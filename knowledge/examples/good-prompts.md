# Good prompts

## Good characteristics
- clear outcome
- explicit context
- defined output format
- stated assumptions or constraints
- realistic scope

## Example — issue intake
Turn these notes into a GitHub issue.
Use `knowledge/policies/delivery-policy.md` and `knowledge/playbooks/pr-review-expectations.md`.
Output sections: title, problem, goals, non-goals, acceptance criteria, dependencies, risks, rollout.

## Example — status update
Summarize these notes for leadership.
Output: summary, what changed, blockers, risks, decisions needed, next steps.

## Example — context capture
Create a Markdown sidecar summary from this policy note set.
Update the glossary if there are durable new terms.
Recommend whether it belongs in a shared Space.
