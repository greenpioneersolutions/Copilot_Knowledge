# Backlog breakdown playbook

## When to use
Use when an initiative is still too large or vague for good execution.

## Inputs
- rough initiative statement
- business or technical goal
- dependencies if known
- risks if known

## Output
Break work into:
- initiative / epic
- features
- tasks
- dependencies
- rollout checkpoints

## Good pattern
1. define the problem
2. define the smallest useful outcome
3. identify non-goals
4. break into reviewable slices
5. identify cross-team dependencies
6. define success signals

## Avoid
- giant umbrella issues
- tasks without acceptance criteria
- hidden rollout work
