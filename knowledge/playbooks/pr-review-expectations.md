# Pull request review expectations

## Purpose
Help managers and engineers maintain review quality.

## Expectations
- work should be scoped for review
- acceptance criteria should be traceable
- rollout or release considerations should be visible when relevant
- tests or validation should be described
- major tradeoffs should be captured outside the PR if they matter later

## Manager view
A manager should be able to answer:
- what changed
- why it changed
- what risk it introduces
- how it will be validated
- what still needs attention
