# Glossary

## Copilot Space
A curated context bundle in GitHub Copilot that can include repositories, issues, pull requests, notes, uploads, transcripts, and other supporting content.

## Durable memory
Information that should outlive a single chat session and be stored in the repository in Markdown.

## Sidecar summary
A Markdown summary created from a raw source artifact such as a PDF, policy, transcript, or meeting note set.

## Repo-wide instructions
A `.github/copilot-instructions.md` file that applies stable guidance across the repository.

## Path-specific instructions
`*.instructions.md` files that apply guidance to specific file patterns.

## Prompt file
A reusable `.prompt.md` file for a repeatable task.

## Custom agent
A specialized Copilot persona defined in a `.agent.md` file.

## Skill
A folder-based capability with a `SKILL.md` file that can load instructions, resources, and supporting assets for multi-step work.

## MCP
Model Context Protocol, used to connect Copilot to external tools and systems.

## Copilot Memory
Repository-level learned memory generated by Copilot, useful but not a substitute for human-curated Markdown memory.

## Manager lane
The lowest-friction enablement path, typically centered on GitHub.com and Spaces.

## Champion lane
The advanced enablement path for technical managers and early adopters who can support others.

## Markdown-first
A design choice that uses Markdown as the primary durable storage format for reusable knowledge.
