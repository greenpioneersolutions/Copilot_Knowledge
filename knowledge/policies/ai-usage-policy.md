# AI usage policy

> Draft starter. Replace with your organization’s real policy.

## Purpose

Define how this team uses AI assistance responsibly.

## Allowed uses

- drafting issues
- summarizing notes
- creating onboarding plans
- generating first-pass documentation
- structuring status updates
- assisting with planning and issue refinement

## Requires review

- all outputs used for important decisions
- all code changes
- all policy interpretations
- all customer-facing or regulatory content

## Sensitive data guidance

Before committing or attaching content:
- verify the source is allowed in Git
- remove sensitive information when required
- prefer summaries over raw document storage
- store raw originals externally if required by policy

## Durable memory rule

If information should outlive a single chat:
- summarize it into Markdown
- store it in the appropriate location in `knowledge/`
- update glossary / FAQ / decisions when appropriate

## Escalation questions

Escalate before using AI on:
- restricted HR material
- legal or compliance-sensitive content
- credentials or secrets
- private security incident details
