# Light CLI cheat sheet

Use this only inside VS Code's integrated terminal until it feels normal.

## Start
```bash
copilot
```

## Mention a file
```text
@README.md summarize setup and dependencies
@docs/architecture.md explain this system for a manager
```

## Run research
```text
/research Compare the deployment flow described in this repo with the current PRs and likely release risks
```

## Switch models when the task changes
```text
/model
```

## Good Level 2 CLI use cases

- project overview
- compare docs vs code reality
- prepare a technical briefing
- investigate a focused repo question

## Avoid in Level 2

- broad autonomous code changes
- blanket permission approval
- tasks you do not plan to review carefully
