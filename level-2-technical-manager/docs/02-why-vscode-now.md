# Why VS Code now

VS Code is not the first step. It is the second useful step.

## Why it belongs here

VS Code adds three things that matter for managers who need deeper context:
- Ask, Plan, and Edit modes
- stronger workspace and file grounding
- shared custom agents inside the workspace

## Why it did not belong in Level 1

For a new user, VS Code introduces too many simultaneous concepts:
- workspace
- file references
- editor context
- chat modes
- custom agents
- terminal

That is not a beginner experience.

## Why CLI appears here too

The terminal is still intimidating for many managers.

So in Level 2, CLI stays small and practical:
- open `copilot`
- mention a file with `@`
- use `/research` for deeper investigation
- switch models only if needed

That is enough to get real value without turning the learner into a CLI power user.
