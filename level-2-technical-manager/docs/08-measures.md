# Measures

## Good Level 2 measures

### Workflow quality
- better issue breakdown quality
- better rollout plans
- better leadership briefs grounded in repo context

### Time
- time to prepare a technical briefing
- time to turn rough asks into delivery plans
- time to produce a useful status update

### Capability growth
- can the learner pick Ask vs Plan correctly?
- can the learner choose the right agent?
- can the learner use the CLI for a focused question without getting lost?

## Avoid weak measures

Do not measure Level 2 success only with:
- raw PR throughput
- generic usage counts
- number of prompts sent

Look for workflow outcomes instead.
