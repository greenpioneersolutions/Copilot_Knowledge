# VS Code chat cheat sheet

## Modes to learn

### Ask mode
Use when you want:
- explanation
- summary
- technical translation
- questions answered without changing code

### Plan mode
Use when you want:
- implementation plans
- rollout plans
- acceptance criteria
- structured next steps

### Edit mode
Use when you want:
- controlled edits to a limited set of files
- small documentation updates
- focused cleanup

## Default rule

Start in **Ask** or **Plan**.
Use **Edit** only when the requested changes are small and reviewable.

## Helpful context moves

- open the relevant file before asking
- reference files directly when needed
- keep the request specific
- ask for facts vs assumptions when the output affects decisions

## Manager prompts that work well

- `Explain this module in plain English.`
- `Create a delivery plan for this change.`
- `What risks should a manager care about here?`
- `Turn this into a weekly status update.`
- `What decisions are still missing?`
