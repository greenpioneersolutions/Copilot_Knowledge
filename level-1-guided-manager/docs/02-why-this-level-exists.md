# Why this level exists

The earlier design started too far up the curve.

It assumed the learner already knew:
- why repo context matters
- how GitHub and VS Code differ
- when custom agents help
- why Markdown memory beats chat history
- how to judge model and tool choices

That is not how most managers start.

## The reset

Level 1 assumes none of that.

Instead, it gives a learner:
- one place to go
- three useful jobs-to-be-done
- one review habit

## Why GitHub.com first

GitHub.com is the lowest-friction surface that still gives real engineering value.

A beginner can:
- chat in repo context
- use a shared Space
- create an issue from rough notes
- understand a PR faster

That is enough to create trust.

## Why not agents yet

Agents add choices:
- which agent
- what tools
- what mode
- what instructions
- whether the agent should edit anything

Those are valuable choices later. They are distracting choices now.

## The design target

The target is not "teach all of Copilot."

The target is:

> remove choices and create one fast win
