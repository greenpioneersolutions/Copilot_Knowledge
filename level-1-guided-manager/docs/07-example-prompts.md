# Example prompts

## Understand the repo
- `Explain this repo in plain English for a manager.`
- `What is changing most right now and why?`
- `Where is delivery risk likely to exist in this workstream?`

## Shape an issue
- `Turn these notes into a clear issue with acceptance criteria and open questions.`
- `Rewrite this ask so engineering can act on it.`
- `Make this issue smaller and more testable.`

## Understand a PR
- `Summarize this PR for a manager.`
- `What is the likely user or operational impact of this change?`
- `What are the best review questions to ask here?`

## Improve weak output
- `Separate fact from assumption.`
- `List what is still unclear.`
- `Rewrite this in plain English.`
- `Give me only the top three risks.`
