# Shared Space setup

## Goal

Create one shared Space that gives managers a safe and useful starting point.

## Use one Space per team or workstream

Do not create a giant enterprise Space.

Good examples:
- Payments team
- Identity platform
- Mobile release train
- Search workstream

Bad examples:
- all engineering
- every repository in the org
- all PDFs from every department

## What to add

Start with:
- one or two core repositories
- a few high-signal issues or PRs
- a short team context note
- a current priorities note
- one glossary note
- one onboarding note

## What to avoid

Avoid:
- giant dumps of documents
- stale notes
- old incident detail that no one needs
- duplicate sources that say the same thing

## The working rule

If a source does not help one of the three plays, it probably does not belong in the Level 1 Space.
