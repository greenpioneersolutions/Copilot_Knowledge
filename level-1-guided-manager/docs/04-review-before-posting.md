# Review before posting

Use this checklist before you submit an issue, comment, or summary.

## Review checklist

1. Is the problem statement actually correct?
2. Did Copilot invent anything important?
3. Are the risks and dependencies real?
4. Is the wording appropriate for the audience?
5. Would I stand behind this if my name stayed on it?

## What to do if the output is weak

- narrow the request
- add more concrete context
- tell Copilot what is missing
- ask it to separate fact from assumption
- ask for open questions instead of guesses

## Good correction prompts

- `Separate facts from assumptions.`
- `Rewrite this for a manager audience in plain English.`
- `What is still unclear or unsupported here?`
- `Give me the top three risks only.`
- `Turn the assumptions into explicit open questions.`
