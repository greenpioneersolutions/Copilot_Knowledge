# Core problems we are solving

These are the core manager pain points this level is designed to address.

## 1) Finding answers and context
Managers lose time hunting through repos, issues, PRs, notes, and prior decisions.

### Response
- shared Space
- Manager Analyst
- short Markdown memory files

## 2) Communication overhead
A large share of manager time goes to status updates, stakeholder notes, and translation across audiences.

### Response
- Communications Writer
- reusable status and release templates
- grounded communication from repo context instead of memory alone

## 3) Planning and issue shaping
Poorly shaped work creates confusion, rework, and weak delegation.

### Response
- Delivery Planner
- issue templates
- explicit assumptions and open questions

## 4) Decision capture
Important context disappears into meetings and chat threads.

### Response
- Decision Logger
- decision log template
- capture useful outputs back into Markdown

## 5) Tactical work stealing manager attention
Managers can become bottlenecks if every bounded engineering task needs manual orchestration.

### Response
- built-in Copilot coding agent for well-scoped work only
- manager review stays focused on scope, risk, and quality
