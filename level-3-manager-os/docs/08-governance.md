# Governance

## Shared rules

### Keep instructions short
Repository and organization instructions should be short, self-contained, and broadly applicable.

### Avoid conflicts
Do not pile up contradictory personal, repository, and organization instructions.

### Prefer Markdown for durable memory
If the output matters later, save it back to Markdown in a known location.

### Keep Spaces curated
A giant Space is a weak Space.

### Review all AI output
Managers are still responsible for:
- issue quality
- communications
- decisions
- delegated work review

## CLI safety

If you use Copilot CLI:
- trust the directory before you start
- review file edits before approval
- review risky commands before approval
- avoid blanket permission approval unless you truly understand the consequences

## Coding-agent safety

Use the built-in coding agent only when:
- the issue is well scoped
- someone will review the PR
- the team understands how follow-up happens on the PR

## Measurement caution

Do not over-trust raw Copilot dashboard numbers for this manager program.

Use a blended scorecard:
- workflow quality
- repeated-question reduction
- status drafting time saved
- decision log capture rate
- selected GitHub metrics
