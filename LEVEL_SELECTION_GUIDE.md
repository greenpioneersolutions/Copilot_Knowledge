# Level selection guide

Use this guide before assigning a level.

## Send someone to Level 1 if...

- they are new to GitHub Copilot
- they mostly use GitHub.com
- they need help reading repos, issues, and PRs
- they get overwhelmed by too many options
- they are not comfortable in VS Code or a terminal

## Send someone to Level 2 if...

- they already got value from Level 1
- they open VS Code sometimes
- they need better technical summaries, plans, or rollout notes
- they can review output critically
- they can handle a small amount of terminal use inside VS Code

## Send someone to Level 3 if...

- they are already productive in Level 2
- they want repeatable team workflows
- they want to teach others
- they are willing to maintain a small shared agent catalog
- they can scope work well enough to delegate bounded tasks safely

## Keep someone out of this program if...

They rarely work in GitHub at all.

Those users usually need Microsoft 365 Copilot more than GitHub Copilot. If their work is mostly email, docs, meetings, and internal search, the better first step is Microsoft 365 Copilot.

## Fast assessment

Score each person from 0 to 2 on each line.

- Uses GitHub.com weekly
- Reads or comments on issues weekly
- Reads or comments on PRs weekly
- Comfortable opening VS Code
- Comfortable asking technical questions about a repo
- Comfortable reviewing AI output before using it
- Comfortable using a terminal

### Suggested placement

- **0 to 5** → Level 1
- **6 to 10** → Level 2
- **11 to 14** → Level 3

Use judgment. This is a placement aid, not a law.
